Operating Systems Terminology

Term Definition
Operating System software packages that coordinate a computer’s internal activities and oversee its communication with the outside world
Job queue data structure determining execution order of jobs residing in mass storage
FIFO objects are removed from the job queue in the order in which they arrived
Interactive processing feature in an operating system that allows a program being executed to carry on a dialogue with the user through remote terminals
Time-sharing operating systems that provide service to multiple users at the same time
Multiprogramming method of time-sharing where time is divided into intervals and then the execution of each job is restricted to only one interval at a time. At the end of each interval, the current job is temporarily set aside and another is allowed to execute during the next interval
Application software consists of the programs for performing tasks particular to the machine’s utilization
System software In contrast to application software, system software performs those tasks that are common to computer systems in general. In a sense, the system software provides the infrastructure that the application software requires
Utility software a class of system software units composed of programs for performing activities that are fundamental to computer installations but not included in the operating system
Graphical user interface (GUI) modern user interfaces where objects to be manipulated, such as files and programs, are represented pictorially on the display as icons and users can issue commands using input devices
Shell older user interfaces that communicated with users through textual messages using a keyboard and monitor screen
Kernel the internal part of an operating system that contains those software components that perform the very basic functions required by the computer installation
Device driver software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine
File manager coordinates the use of the machine’s mass storage facilities; maintains records of all the files stored in mass storage, including where each file is located, which users are allowed to access the various files, and which portions of mass storage are available for new files or extensions to existing files
Memory manager part of the operating system that coordinates the system’s usage of main memory
Virtual manager large “fictional” memory space created by paging
Boot strapping performed by a computer each time it is turned on; procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory (which is essentially empty when the machine is first turned on)
ROM read-only memory, a small portion of a computer’s main memory where the CPU expects to find its initial program is constructed from special nonvolatile memory cells
Deadlock the condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another
Spooling technique of holding data for output at a later but more convenient time
Super user (or administrator) person with highly privileged access to the operating system by identifying him- or herself as the administrator (usually by name and password) during the login procedure. Once this contact is established, the administrator can alter settings within the operating system, modify critical software packages, adjust the privileges granted to other users, and perform a variety of other maintenance activities that are denied normal users
Multi-factor authentication security solution against weak passwords that requires more than one piece of evidence to gain access
